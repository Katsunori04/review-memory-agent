# Review Memory Agent

## 機能一覧・ユーザー動線まとめ

Review機能の詳細仕様は `docs/review-detailed-design.md` を参照。
Memory機能の詳細仕様は `docs/memory-detailed-design.md` を参照。
Apply機能の詳細仕様は `docs/apply-detailed-design.md` を参照。

## 1. プロダクトコンセプト

**Review Memory Agent** は、正式レビュー前のセルフレビューを支援し、正式レビューコメント・会議の文字起こし・観点メモを次回以降のレビュー観点として活用するAIエージェント。

一言でいうと、

> **資料はAIがセルフレビューする。
> 人間の指摘や会議の知見は、AIがレビューMemoryに変える。
> 次回は、そのMemoryを踏まえてレビューする。**

---

# 2. コア体験

このプロダクトの体験は、以下の3つに集約する。

## Review

資料をアップロードすると、AIが文脈を推定し、セルフレビューする。

## Memory

正式レビューコメント・会議文字起こし・観点メモから、AIが登録すべき論点を抽出する。

## Apply

抽出された論点をレビュー方針カードとして保存し、次回以降のセルフレビューに反映する。

---

# 3. ユーザー動線

## 動線A：AIセルフレビュー

### 目的

正式レビューに出す前に、AIで資料の不足論点・リスク・修正案を確認する。

```text
1. ホームで「AIセルフレビューを開始」を押す
2. AIセルフレビュー画面で資料をアップロードする
3. AIが資料タイプ・レビュー目的・重点観点を自動推定する
4. ユーザーが必要に応じて推定結果を修正する
5. AIセルフレビューを実行する
6. レビュー通過可能性スコアを確認する
7. 指摘カード・修正案・追記文案を確認する
8. 資料改善に活用する
```

### ポイント

* レビュータイプや目的は最初から手入力しない
* AIが自動推定する
* ただし、ユーザーが修正できる
* 正式レビュー前のセルフチェックに特化する

---

## 動線B：レビューMemory登録

### 目的

正式レビューコメントや会議の知見を、次回以降に使えるレビュー観点として登録する。

```text
1. ホームで「レビューMemoryを登録」を押す
2. 正式レビューコメント、会議文字起こし、観点メモを貼り付ける
3. AIが登録候補となる論点を抽出する
4. ユーザーが登録したい論点を選ぶ
5. AIがレビュー方針カードを生成する
6. ユーザーが保存する
7. 次回以降のAIセルフレビューに反映される
```

### 入力できるもの

| 入力種別       | 例                |
| ---------- | ---------------- |
| 正式レビューコメント | 上司・レビュワーから受けた指摘  |
| 会議文字起こし    | レビュー会・設計レビュー会の発言 |
| 観点メモ       | 今後レビュー時に見たい観点    |
| チャットコメント   | TeamsやSlack上の指摘  |
| QA・問い合わせ回答 | 情シス・法務などの見解      |
| 差し戻し理由     | 稟議・提案書が戻された理由    |

---

## 動線C：次回セルフレビューへの反映

### 目的

過去に登録した正式レビューコメントや観点メモを、次回のAIレビューに活かす。

```text
1. 新しい資料をアップロードする
2. AIが資料タイプ・目的・重点観点を自動推定する
3. 保存済みレビュー方針カードを検索する
4. 類似する方針をレビューに反映する
5. 適用した過去方針を表示する
6. 前回登録した観点を踏まえた指摘が出る
```

### 表示例

```text
前回のレビュー会メモに基づき、以下の観点を重点確認しました。

- PoC提案では、測定方法と評価タイミングを明記する
- 生成AI提案では、入力禁止情報とログ管理方針を記載する
```

| 15 | 適用した過去方針表示   | 今回使ったMemoryを表示       |

---

## Should：あると分かりやすい機能

| No | 機能            | 内容                |
| -: | ------------- | ----------------- |
|  1 | カテゴリ自動分類      | 効果、セキュリティ、運用などに分類 |
|  2 | 登録推奨度表示       | どの論点を保存すべきか表示     |
|  3 | 方針カード一覧       | 保存済みMemoryを一覧表示   |
|  4 | AIだけでは判断できない点 | 人間確認が必要な点を表示      |
|  5 | レビュー厳しさ選択     | 軽め・標準・厳しめを選択      |

---

## Cut：MVPでは不要な機能

| 機能                | 理由                    |
| ----------------- | --------------------- |
| 再セルフレビュー          | 画面遷移と説明が増える           |
| 改善差分表示            | コンセプトの核ではない           |
| 提出パッケージ生成         | 今回はセルフレビューとMemory化に集中 |
| レビュワー指定           | レビュワーをシステムに巻き込まない     |
| 通知・期限管理           | 業務管理色が強くなる            |
| Teams / Outlook連携 | 将来機能でよい               |
| 本格的なカード編集         | デモでは生成・保存で十分          |
| ダッシュボード           | 将来機能でよい               |

# 7. 最終的な機能コンセプト

## Review

資料を入れるだけで、AIが文脈を推定し、セルフレビューする。
推定結果は人間が修正できる。

## Memory

正式レビューコメント・会議文字起こし・観点メモから、AIが登録すべき論点を抽出する。
人間が登録する論点を選択する。

# 8. MVPのゴール

MVPで伝えるべきことは、以下の1点です。

> **AIが資料をレビューするだけでなく、人間のレビュー観点や会議の知見をMemory化し、次回以降のレビューに活かせる。**

これが伝われば、ハッカソンのコンセプトとしてかなり強いです。
